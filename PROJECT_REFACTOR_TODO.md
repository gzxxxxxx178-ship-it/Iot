# 智慧农业 IoT 项目改造问题清单

## 使用规则

- 本文件是项目改造的唯一问题台账，用于记录范围、优先级、状态和验收标准。
- 默认只维护台账，不自动实施任务。用户明确指定任务编号后，一次只执行一个任务。
- 每项任务完成后必须经过验证，更新本文件状态，并按项目规范单独提交 Git。
- 不得把 Wi-Fi 密码、数据库密码、JWT 密钥、API Key、OAuth 密钥或支付密钥提交到 Git。

## 状态说明

| 状态 | 含义 |
|---|---|
| `待执行` | 问题已确认，尚未开始修改 |
| `执行中` | 已获得用户指令，正在修改和验证 |
| `受阻` | 缺少必要信息或外部条件，暂时不能完成 |
| `已完成` | 修改、验证、台账更新和 Git 提交均已完成 |

## 建议执行顺序

```text
P0-01 → P0-02 → P0-03 → P0-04
      → P1-01 → P1-02 → P1-03 → P1-04
      → P2-01 → P2-02 → P2-03 → P3-01
```

## P0：直接影响当前功能或凭据安全

### P0-01 修复前端 axios 二次解包

- **状态**：`已完成`
- **问题**：`src/api/request.js` 已把后端 `ApiResponse` 解包为 `body.data`，多个 API 函数又执行 `.then(res => res.data)`，使成功响应变成 `undefined`。
- **影响**：登录、注册、设备列表、历史数据、聊天、支付、仪表盘和报警请求可能无法获得有效返回值。
- **范围**：`vue/IoT/src/api/request.js`、`vue/IoT/src/api/*.js`，以及必要的前端测试。
- **验收标准**：统一只解包一次；各 API 返回真实业务数据；前端生产构建通过；关键 API 有回归验证。
- **完成结果**：移除 17 处二次解包；新增 `test:api-contract` 契约测试；2 项测试和生产构建均通过。
- **执行命令示例**：`执行 P0-01`

### P0-02 补齐仪表盘后端接口

- **状态**：`已完成`
- **问题**：前端调用 `/api/dashboard/stats` 和 `/api/dashboard/device-status`，后端没有对应 Controller 和 Service。
- **影响**：仪表盘统计请求失败，设备数量、在线数量和平均温湿度不能可靠展示。
- **范围**：Java Controller、Service、Repository、DTO、测试及相关前端调用。
- **方法约束**：在线状态必须依据设备最后上报时间判断，不能把 `linkage` 报警字段当作在线状态。
- **验收标准**：两个接口返回统一响应结构；统计口径明确；空数据和多设备场景有测试；后端测试与前端构建通过。
- **完成结果**：新增统计和状态分布接口；在线阈值默认 30 秒且可配置；平均温湿度只聚合在线设备最新值；前端仪表盘已接入真实统计。
- **执行命令示例**：`执行 P0-02`

### P0-03 实现报警规则与报警记录后端

- **状态**：`已完成`
- **问题**：前端已调用 `/api/alarm/rules` 和 `/api/alarm/records`，后端没有对应实体、存储和接口。
- **影响**：报警页面不能形成真实闭环，规则无法持久化，传感器越限也不会生成可追溯记录。
- **范围**：报警规则实体、报警记录实体、Repository、Service、Controller、DTO、测试及前端联调。
- **方法约束**：阈值单位、比较运算、启停状态、设备范围和重复报警抑制必须明确。
- **验收标准**：规则可增删改查；数据越限能生成记录；重复报警策略可验证；接口和数据库测试通过。
- **完成结果**：新增规则和报警记录持久化、增删改查及记录查询接口；传感器入库后自动执行温度/湿度/水位 ADC 越限判断；支持设备范围、启停状态和 0–86400 秒冷却抑制；前端已支持规则编辑和完整字段配置。
- **执行命令示例**：`执行 P0-03`

### P0-04 隔离 Arduino Wi-Fi 凭据

- **状态**：`已完成`
- **问题**：Wi-Fi 名称和密码硬编码在受 Git 管理的固件中，存在凭据泄露风险。
- **影响**：仓库、历史提交、终端输出或截图可能暴露网络凭据。
- **范围**：`arduino/MQTT/MQTT.ino`、本地 `secrets.h`、可提交的 `secrets.example.h` 和最小范围 `.gitignore` 规则。
- **方法约束**：真实凭据只保存在本地忽略文件；已经暴露的密码需要由用户在路由器或热点端更换。
- **验收标准**：固件可正常编译和烧录；Git 跟踪内容不含真实凭据；示例配置能够指导重新部署。
- **完成结果**：固件改为从本地 `secrets.h` 读取 Wi-Fi 配置；新增可提交的 `secrets.example.h` 和精确忽略规则；ESP8266 NodeMCU 编译、写入校验及硬复位成功。历史中已经出现的 Wi-Fi 密码仍需在路由器或热点端更换。

## P1：通信安全与业务闭环

### P1-01 增加 WebSocket 鉴权与来源限制

- **状态**：`已完成`
- **问题**：`/ws/**` 绕过 Spring Security，服务端允许任意来源；前端注释称携带 Token，但实际连接 URL 没有 Token。
- **影响**：未认证客户端可能订阅实时传感器数据。
- **范围**：Spring WebSocket 握手拦截器、安全配置、前端 WebSocket composable 和测试。
- **验收标准**：无效或缺失 Token 的握手被拒绝；合法用户可以连接；生产环境来源采用白名单；自动重连仍可用。
- **完成结果**：新增受JWT保护的一次性票据接口，票据默认30秒有效且只能消费一次；握手依次校验Origin白名单和票据，移除WebSocket对安全过滤器链的绕过；前端断线后重新领取票据并自动重连。生产部署需将 `websocket.allowed-origins` 设置为实际Cloudflare Pages来源。
- **执行命令示例**：`执行 P1-01`

### P1-02 提升 MQTT 安全性与可靠性

- **状态**：`已完成（硬件在线复验按用户要求跳过）`
- **问题**：当前使用公共 Broker、1883 明文连接和固定 Java 客户端 ID，断线后没有自动重连。
- **影响**：存在窃听、伪造消息、客户端互踢和网络波动后永久离线风险。
- **范围**：Java MQTT 配置与服务、ESP8266 固件、环境配置和联调测试。
- **方法约束**：优先采用 TLS、唯一客户端 ID、指数退避重连、遗嘱消息和设备级 Topic；部署成本需与当前原型阶段匹配。
- **验收标准**：断网恢复后自动重连并重新订阅；并行设备不互踢；生产凭据不进入 Git；控制和数据 Topic 边界明确。
- **完成结果**：VPS 部署私有 Mosquitto TLS 8883 和项目 IoT CA；后端与 ESP8266 使用唯一客户端 ID、QoS 1、遗嘱消息和最高 60 秒指数退避，连接恢复后自动订阅。凭据分别保存在 macOS 钥匙串、VPS 600 权限配置和本地忽略文件。固件编译、写入和校验成功；当前手机热点链路的设备在线确认经用户要求跳过，需在固定 Wi-Fi 下复验。
- **执行命令示例**：`执行 P1-02`

### P1-03 增加设备身份认证与 Topic ACL

- **状态**：`已完成`
- **问题**：当前设备和服务端没有可靠的发布、订阅权限边界，第三方可能伪造传感器数据或控制命令。
- **影响**：数据真实性和设备控制安全无法保证。
- **范围**：Broker 账号与 ACL、设备身份字段、后端校验和部署说明。
- **验收标准**：设备只能发布自身数据并订阅自身控制 Topic；后端拒绝越权设备消息；凭据支持独立轮换。
- **完成结果**：Broker 为后端和 `device001` 配置独立账号、密码库与最小 ACL；Topic 统一为 `agri/{deviceId}/data|status|control`。后端同时校验 Topic 结构与载荷设备 ID，控制接口只接受设备 ID 和动作白名单；正向 TLS 收发及反向越权不投递测试通过。
- **执行命令示例**：`执行 P1-03`

### P1-04 自动化规则持久化与执行闭环

- **状态**：`已完成`
- **问题**：自动化页面目前是前端 Mock，刷新后规则丢失，也没有服务端执行引擎。
- **影响**：条件触发和设备动作只是界面演示，不能用于实际控制。
- **范围**：规则实体、Repository、Service、Controller、条件求值、动作执行记录和前端联调。
- **方法约束**：规则执行必须包含防抖、冷却时间、失败记录和手动停用机制，避免传感器抖动导致设备频繁启停。
- **验收标准**：规则可持久化；满足条件时只触发预期动作；异常和重复触发可追溯；停用规则不会执行。
- **完成结果**：新增自动化规则与执行记录表、CRUD 接口和服务端条件引擎；支持设备范围、连续命中防抖、动作冷却、手动停用及成功/失败审计。启动、停止动作通过设备级 MQTT Topic 下发；通知动作当前记录为站内审计事件。前端已改为真实接口并展示最近 100 条执行记录。
- **执行命令示例**：`执行 P1-04`

## P2：数据质量与工程稳定性

### P2-01 完善历史数据查询与治理

- **状态**：`已完成`
- **问题**：历史数据缺少完整的设备筛选、参数边界、异常值标记、保留周期和面向数据增长的索引设计。
- **影响**：数据量增加后查询性能下降，异常传感器值可能直接进入统计和图表。
- **范围**：数据库索引、Repository 查询、分页 DTO、参数校验、异常值策略和测试。
- **验收标准**：支持设备和时间范围组合查询；分页参数有上限；关键查询使用合适索引；异常值不会污染核心统计。
- **完成结果**：历史接口支持设备与时间组合筛选，分页上限为 100 条、单次范围最长 366 天；新增设备/接收时间和接收时间/质量状态索引。新数据按当前 DHT11 与 ESP8266 采集边界（温度 0–50 ℃、湿度 0–100%、水位 0–1023 ADC、RSSI -120–0 dBm）标记质量，异常原始记录保留但不参与报警、自动化和仪表盘均值。默认每日批量清理 365 天前数据，周期可配置。历史页面展示质量状态，并可按当前设备和时间条件导出最多 50000 条 UTF-8 CSV，而非仅导出当前分页。
- **执行命令示例**：`执行 P2-01`

### P2-02 隔离后端测试环境

- **状态**：`已完成`
- **问题**：后端测试会加载 `dev` Profile，并实际连接开发数据库和公共 MQTT Broker。
- **影响**：测试依赖外部网络和真实配置，可能污染数据，结果也不稳定。
- **范围**：`application-test.properties`、测试注解、数据库替身或 Testcontainers、MQTT Mock 和 CI 配置。
- **验收标准**：离线环境可运行单元测试；测试不连接开发或生产数据库与 Broker；20 个现有测试继续通过。
- **完成结果**：新增 `test` Profile 和 H2 内存数据库，`@SpringBootTest` 显式使用测试 Profile；生产 MQTT、Redis 配置在测试 Profile 下禁用并替换为 Mockito 替身，测试配置不包含真实凭据或生产地址；新增 GitHub Actions 离线测试工作流。当前 83 个测试全部通过。
- **执行命令示例**：`执行 P2-02`

### P2-03 统一 Git 仓库边界

- **状态**：`已完成`
- **问题**：根目录、`java/` 和 `vue/IoT/` 分别存在 Git 元数据，根仓库干净不代表两个内层仓库干净。
- **影响**：容易遗漏改动、提交到错误远端，或使不同仓库历史产生冲突。
- **范围**：仓库边界决策、远端备份、嵌套 `.git` 处理、文档和提交工作流。
- **风险约束**：删除或迁移任何 Git 元数据前必须单独获得用户确认，并验证所有历史和未提交改动均有备份。
- **验收标准**：形成唯一且明确的版本管理方案；所有现有改动可追溯；开发和部署文档与方案一致。
- **完成结果**：确认根目录 GitHub 仓库为唯一版本库；已将 `java/.git` 和 `vue/IoT/.git` 的历史对象、状态、差异和未跟踪文件清单备份至工作区外，并移出工作区；根仓库状态保持干净。根目录及 Java、Vue 文档已统一 Git 操作和部署提交边界，未自动推送。
- **执行命令示例**：`执行 P2-03`

### P2-04 完善设备管理与生命周期

- **状态**：`已完成`
- **问题**：设备列表由传感器历史记录临时反推，没有独立设备档案，无法可靠执行新增、编辑、停用、归档和恢复。
- **影响**：设备名称、类型和部署位置无法持久化；直接删除历史数据会破坏报警、自动化和统计追溯。
- **范围**：设备实体、Repository、生命周期 Service、CRUD 接口、遥测与控制状态校验、前端设备管理工作台和测试。
- **验收标准**：支持设备注册、查询、编辑、启停、归档和恢复；设备ID创建后不可修改；归档不删除历史数据；停用或归档设备不能接收遥测或发送控制指令。
- **完成结果**：新增 `iot_devices` 独立设备档案表与完整生命周期接口；可信 MQTT 设备首次上报或升级前历史设备会自动补录。设备页支持注册、编辑、启停、软删除归档、恢复、10秒状态刷新和远程控制；在线状态统一依据最近30秒上报判断。注册档案不自动创建 Broker 凭据，设备仍需配置独立 MQTT 账号与 ACL。
- **执行命令示例**：`执行 P2-04`
